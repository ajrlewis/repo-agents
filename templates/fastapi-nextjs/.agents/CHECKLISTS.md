# Checklists

## Before finishing

- [ ] Branch created and scoped to one feature/fix
- [ ] Change set kept small and focused
- [ ] Unit tests added/updated for new behavior
- [ ] Regression tests run for impacted backend/frontend areas
- [ ] Docker dev container/image updated if relevant
- [ ] Docker production container/image updated if relevant
- [ ] Commands/docs updated if behavior changed
- [ ] PR notes include what changed, tests run, and risk/rollback notes
